# Sidekick — Server Deployment Guide

Deploy the payment + license server to Railway in ~15 minutes.

---

## What the server does

| Endpoint | Purpose |
|---|---|
| `POST /api/validate` | Check tier + usage count for a device |
| `POST /api/checkout` | Create Stripe checkout session (optional — app uses payment links instead) |
| `POST /stripe/webhook` | Receive payment → generate HMAC key → email to customer |
| `GET /health` | Railway health check |

The app works **locally without the server** (ownerMode=true or valid HMAC key). You only need the server running for the **purchase → key email** flow to work for paying users.

---

## 1 · Prerequisites

- Railway account → [railway.app](https://railway.app) (free tier is enough)
- Stripe account → [stripe.com](https://stripe.com)
- Resend account → [resend.com](https://resend.com) (100 emails/day free)
- Your domain or a Resend subdomain for `EMAIL_FROM`

---

## 2 · Generate LICENSE_SECRET

This must be **identical** in the server AND in `app.config.js`. Use the same value you already have in `app.config.js`:

```bash
# Check what you have:
cat app.config.js | grep licenseSecret
```

Keep it secret. If it leaks, regenerate and redeploy.

## 4 · Deploy to Railway

```bash
cd server
# Install Railway CLI if needed:
brew install railway

# Login and deploy:
railway login
railway init        # create new project
railway up          # deploy
```

Or connect via GitHub repo in Railway dashboard (auto-deploys on push).

---

## 5 · Set environment variables in Railway

In Railway dashboard → your service → Variables, add:

```
ANTHROPIC_API_KEY     = sk-ant-api03-...
STRIPE_SECRET_KEY     = sk_live_...
STRIPE_WEBHOOK_SECRET = whsec_...   (set after step 6)
STRIPE_PRICE_PRO_MONTHLY  = price_...
STRIPE_PRICE_PRO_YEARLY   = price_...
STRIPE_PRICE_MAX_MONTHLY  = price_...
STRIPE_PRICE_MAX_YEARLY   = price_...
RESEND_API_KEY        = re_...
EMAIL_FROM            = Sidekick <hi@yourdomain.com>
LICENSE_SECRET        = (same 64-char hex as app.config.js licenseSecret)
APP_URL               = https://your-service.up.railway.app
FREE_MODEL            = claude-haiku-4-20251001
PRO_MODEL             = claude-sonnet-4-20250514
MAX_MODEL             = claude-sonnet-4-20250514
```

## 6 · Configure Stripe webhook

In [Stripe Dashboard → Webhooks](https://dashboard.stripe.com/webhooks) → Add endpoint:

- URL: `https://your-service.up.railway.app/stripe/webhook`
- Events to listen for:
  - `checkout.session.completed`
  - `customer.subscription.deleted`

Copy the **Signing secret** (`whsec_...`) → paste into Railway as `STRIPE_WEBHOOK_SECRET`.

---

## 7 · Verify it works

```bash
# Health check:
curl https://your-service.up.railway.app/health
# → {"ok":true,"uptime":...}

# Test key generation (offline — uses local scripts/gen-keys.js):
LICENSE_SECRET=your_secret node scripts/gen-keys.js
```

Do a test purchase in Stripe test mode → confirm you receive the key email.

## 8 · Match the secret in the app

In `/Users/emiirrr/sidekick/app.config.js` (gitignored, local only):
```js
module.exports = {
  licenseSecret: 'same_64_char_hex_as_railway_env',
  ownerMode: true,   // keep true for your local dev builds
};
```

Keys generated by the server will now validate in any user's app.

## Security notes

- `ownerMode: true` in `app.config.js` — never commit, never push
- `licenseSecret` — never commit, never push
- `STRIPE_WEBHOOK_SECRET` — server-side only, never in app
- App validates keys **locally via HMAC** — no server roundtrip on launch
